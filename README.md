# The ResNet18 CNN architecture#
1. Why residual connections help optimization
   
   In residual connections, we introduce 'skip' connections or short-cut connections. These are simple, no operation connections across layers, that directly output the input value x (identty func). So, instead of re-evaluating features in every block, only the difference between expected output and input is added. If the expected output to be calculated is F(x), the model calculates H(x)=F(x)+x. Or essentially, F(x)=H(x)-x.
   
2.  How skip connections improve gradient flow
   I've already stated that we're computing the output as the sum of our main transformation and a short-cut identity. During backpropagation, this addition means the gradient flowing backward through the block splits into two paths one through the learned transformation F(x), and one directly through the identity (or projection) shortcut. The shortcut path provides a route for gradients to reach earlier layers without being repeatedly multiplied by the weights and activation derivatives of every intermediate layer, which is what causes gradients to vanish in very deep plain networks. Even if the gradient through the main path becomes very small, the shortcut path keeps it at least 1. In the research paper that i read on ResNet, the 152 layer residual network was still less complex than plain cnn.

3. The effect of network depth on performance
In my model (4 stages and 2 blocks/stage),  i reached 93.30% validation accuracy on CIFAR10 after 100 epochs and training accuracy of 99.95%. This level of performance at this depth supports the core argument of the ResNet paper: on ImageNet, the researchers found that with residual learning, their deeper 34-layer resnet outperformed the 18-layer resnet by 2.8%, exhibiting lower training error and better generalization to validation data. This was opposite the result they observed for plain cnn networks of the same depths, where the deeper plain network performed worse. This shows depth itself is not the problem andis only beneficial once the optimization difficulty introduced by stacking many layers is solved trhough residual connections.

4. Challenges encountered during implementation
  The most time taking and difficult challenge imo was correctly handling dimension mismatches at stage transitions. Each stage's first block changes both the number of channels and the the spatial resolution (stride=2), which means the shortcut path can no longer be a simple identity. it required a 1x1 convolution projection to match the output shape before the addition can happen. So i had to understand this logic and make a function so that each block tracks the channel no.

Memory usage: transformers use more memory than LSTMs at equivalent capacity, since attention computes a full (seq_len × seq_len) matrix — at seq_len=72.
Training stability: this can be compared by looking at the convergance of the plots in the repo.
Why attention helps: attention lets every timestep look directly at every other timestep in one operation, with learned weights determining relevance — so the model can learn "what happened 48 hours ago matters more than what happened 2 hours ago for this prediction" explicitly, rather than hoping that information survived 70 sequential hidden state updates.
Differences between recurrence and attention: recurrence compresses history into a fixed-size vector sequentially; attention keeps all timesteps accessible simultaneously and routes information via a learned relevance matrix. Recurrence is O(seq_len) in time and O(1) in memory per step; attention is O(1) in time (parallelizable) but O(seq_len²) in memory.
When LSTMs are still useful: small datasets (transformers need more data to learn good attention patterns), memory-constrained environments, very long sequences where attention memory cost becomes prohibitive, and deployment on edge devices where sequential processing is acceptable.
When transformers perform better: large datasets, tasks with complex long-range dependencies, when training speed matters (parallelism), and when you want interpretability via attention weights.

Why attention helps sequence modeling
A standard recurrent model processes a sequence one step at a time, compressing everything it has seen into a single fixed-size vector before moving to the next step. The problem is that by the time we reach timestep 72, the influence of timestep 1 has been passed through 71 sequential transformations so it may have faded, been overwritten, or distorted beyond recognition. Attention solves this by letting every timestep look directly at every other timestep simultaneously, with no intermediary. At each position, the model computes a relevance score between that position and all others, then builds its output as a weighted combination of all positions based on those scores. So if hour 1's reading is genuinely important for predicting hour 72's temperature, the model can learn to assign it a high attention weight and pull that information forward directly, without it having to survive 71 gate operations intact.

Differences between recurrence and attention
Recurrence is sequential by nature: each step depends on the previous step's output, so we cannot process timestep 5 until timesteps 1 through 4 are done. This makes it inherently slow to parallelize across time. It also compresses all of history into one fixed-size vector, which is the core bottleneck for long sequences. Attention is parallel by nature i.e all timesteps are processed simultaneously, since no step depends on another step's output during the forward pass. This makes transformers significantly faster to train on GPUs, which are built for parallel computation. The tradeoff is memory: attention computes a relevance matrix of size seq_len × seq_len, which grows quadratically with sequence length. At seq_len=72 this is trivially small, but for very long sequences (thousands of timesteps) it becomes the limiting factor. Recurrence, by contrast, uses constant memory per step regardless of sequence length.

Situations where LSTMs may still be useful
Small datasets as transformers need large amounts of data to learn good attention patterns; an LSTM can generalize reasonably well from less data since its inductive bias (sequential processing, gated memory) is already well-suited to time series without needing to learn it from scratch. Memory-constrained deployment an LSTM's per-step memory footprint is fixed and predictable, making it easier to deploy on edge devices or embedded systems. Very long sequences, when sequence length runs into the thousands, the quadratic memory cost of attention becomes prohibitive; an LSTM handles arbitrarily long sequences at constant memory cost per step. Online/streaming prediction — because an LSTM processes one step at a time and maintains a running state, it naturally handles streaming data where you want to update predictions as each new reading arrives, without reprocessing the entire history.

Situations where transformers perform better
Large datasets where the model has enough examples to learn meaningful attention patterns and tasks with complex, non-local dependencies. When training speed matters as the parallelism of attention means transformers train significantly faster per epoch on modern GPUs than an equivalent LSTM, especially a hand-rolled one like ours. When interpretability is desired bcuz attention weights can be visualized directly, showing which past timesteps the model found most relevant for each prediction, giving a window into the model's reasoning that an LSTM's hidden state doesn't provide.
